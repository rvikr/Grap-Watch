# iOS WKWebView Wrapper - Build & Publish Guide

This directory contains the native Swift source code to wrap the **Grap Watch** application for iOS. 

---

## 1. Project Setup in Xcode

Since Xcode projects use complex binary configurations, follow these quick steps to build it on a macOS machine:

1. **Create a New Project**:
   - Open Xcode on a Mac.
   - Choose **File > New > Project...**
   - Select **iOS > App** and click **Next**.
   - Set **Product Name** to `Grap Watch` and **Organization Identifier** to `com.grapwatch` (resulting in Bundle ID `com.grapwatch`).
   - For **Interface**, choose **Storyboard**.
   - For **Language**, choose **Swift**.
   - Save the project inside the `ios/` folder.

2. **Add Files**:
   - Delete the default `ViewController.swift`, `AppDelegate.swift`, and `SceneDelegate.swift` generated by Xcode.
   - Right-click the project folder in Xcode, select **Add Files to "Grap Watch"...**, and select:

     - [SceneDelegate.swift](file:///c:/Users/rk/Documents/Grap%20Watch/ios/GrapWatch/SceneDelegate.swift)
     - [ViewController.swift](file:///c:/Users/rk/Documents/Grap%20Watch/ios/GrapWatch/ViewController.swift)
     - [offline.html](file:///c:/Users/rk/Documents/Grap%20Watch/ios/GrapWatch/offline.html)
     - [Info.plist](file:///c:/Users/rk/Documents/Grap%20Watch/ios/GrapWatch/Info.plist) (or copy keys from it into your Xcode Target Info tab)
   - Ensure **Copy items if needed** is checked and **Create groups** is selected.

## 2. Capabilities & Background Modes

To enable background notifications and shared widget storage:
1. Select the project target in Xcode and go to the **Signing & Capabilities** tab.
2. Click **+ Capability** and add **App Groups**.
   - Add a new App Group container: `group.com.grapwatch`.
   - This matches the suite name inside `ViewController.swift` and is required to share the AQI data with the WidgetKit Home Screen widget.
3. Click **+ Capability** and add **Background Modes**.
   - Check **Background fetch** and **Remote notifications** (if using push notifications in the future).

## 3. Creating the iOS Widget (WidgetKit)

To add the Home Screen widget:
1. In Xcode, select **File > New > Target...**
2. Choose **Widget Extension** and click **Next**.
3. Name it `GrapWatchWidget` and click **Finish** (Decline activating scheme if prompted).
4. Select the `GrapWatchWidget` target, go to **Signing & Capabilities**, click **+ Capability**, and enable the same App Group: `group.com.grapwatch`.
5. Open the generated `GrapWatchWidget.swift` file and replace its contents with the following SwiftUI code to display the widget:

```swift
import WidgetKit
import SwiftUI

struct Provider: TimelineProvider {
    func placeholder(in context: Context) -> SimpleEntry {
        SimpleEntry(date: Date(), aqi: 0, stageName: "Loading...", stageColor: "#ff6b35")
    }

    func getSnapshot(in context: Context, completion: @escaping (SimpleEntry) -> ()) {
        let entry = readSharedData()
        completion(entry)
    }

    func getTimeline(in context: Context, completion: @escaping (Timeline<Entry>) -> ()) {
        let entry = readSharedData()
        // Reload every 30 minutes
        let nextUpdate = Calendar.current.date(byAdding: .minute, value: 30, to: Date()) ?? Date()
        let timeline = Timeline(entries: [entry], policy: .after(nextUpdate))
        completion(timeline)
    }
    
    private func readSharedData() -> SimpleEntry {
        let defaults = UserDefaults(suiteName: "group.com.grapwatch")
        let aqi = defaults?.integer(forKey: "aqi") ?? 0
        let stageName = defaults?.string(forKey: "stageName") ?? "—"
        let stageColor = defaults?.string(forKey: "stageColor") ?? "#ff6b35"
        return SimpleEntry(date: Date(), aqi: aqi, stageName: stageName, stageColor: stageColor)
    }
}

struct SimpleEntry: TimelineEntry {
    let date: Date
    let aqi: Int
    let stageName: String
    let stageColor: String
}

struct GrapWatchWidgetEntryView : View {
    var entry: Provider.Entry
    
    // Parse hex colors safely
    var parsedColor: Color {
        let hex = entry.stageColor.trimmingCharacters(in: CharacterSet.alphanumerics.inverted)
        var int: UInt64 = 0
        Scanner(string: hex).scanHexInt64(&int)
        let r, g, b: UInt64
        switch hex.count {
        case 3: // RGB (12-bit)
            (r, g, b) = ((int >> 8) * 17, (int >> 4 & 0xF) * 17, (int & 0xF) * 17)
        case 6: // RGB (24-bit)
            (r, g, b) = (int >> 16, int >> 8 & 0xFF, int & 0xFF)
        default:
            (r, g, b) = (255, 107, 53) // Fallback #ff6b35
        }
        return Color(red: Double(r) / 255, green: Double(g) / 255, blue: Double(b) / 255)
    }

    var body: some View {
        VStack(spacing: 4) {
            Text(entry.aqi > 0 ? "\(entry.aqi)" : "—")
                .font(.system(size: 40, weight: .bold, design: .rounded))
                .foregroundColor(parsedColor)
            
            Text(entry.stageName)
                .font(.system(size: 11, weight: .semibold))
                .foregroundColor(Color(white: 0.7))
                .multilineTextAlignment(.center)
            
            Text(entry.date, style: .time)
                .font(.system(size: 9))
                .foregroundColor(Color(white: 0.4))
                .padding(.top, 4)
        }
        .padding(12)
        .containerBackground(Color(red: 18/255, green: 21/255, blue: 28/255), for: .widget) // matches #12151c
    }
}

@main
struct GrapWatchWidget: Widget {
    let kind: String = "GrapWatchWidget"

    var body: some WidgetConfiguration {
        StaticConfiguration(kind: kind, provider: Provider()) { entry in
            GrapWatchWidgetEntryView(entry: entry)
        }
        .configurationDisplayName("GRAP Status")
        .description("Displays the current Delhi AQI and GRAP Stage.")
        .supportedFamilies([.systemSmall])
    }
}
```

## 4. Building for Production (App Store Release)

To generate the final `.ipa` production binary:
1. Select **Any iOS Device (arm64)** as the target device in the Xcode toolbar.
2. Choose **Product > Archive**.
3. Once the archiver finishes, the Organizer window will pop up.
4. Select the latest archive and click **Distribute App**.
5. Choose **App Store Connect** for store distribution (or **Ad Hoc** for manual test installs) and follow the prompts to complete the signing, upload, and deployment process.
